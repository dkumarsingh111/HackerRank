<<< QUESTION 1 >>>

What is the additional number of parameters that you will need apart from the pretrained weights in the following scenarios? (a) Text classification (3 classes) using BERT-base, (b) Text Summarization using T5.


### WRITE ANSWER BELOW ###

(a) Text Classification (3 Classes) using BERT-base

BERT-base is a pretrained encoder model with:

- Hidden size = 768
- Output representation for the [CLS] token = 768-dimensional vector

For a 3-class classification problem, we add a classification layer on top of BERT.

The classification layer consists of:

1. Weight matrix: 768×3=2304 parameters
2. Bias vector: 3 parameters

Therefore,

	Additional Parameters=768×3+3
			     =2304+3
			     =2307

Answer: Additional parameters required = 2307


(b) Text Summarization using T5

T5 is an encoder-decoder model designed for text generation tasks such as summarization.

The pretrained T5 model already contains:

- Encoder
- Decoder
- Language modeling head

During fine-tuning for summarization, the same pretrained architecture is used without adding any new output layer.

Therefore:

	Additional Parameters=0

Answer: Additional parameters required = 0

(Only the existing pretrained parameters are fine-tuned.)

<<< QUESTION 3 >>>

Show that the self-attention score in the rotatory embeddings depends on the positional difference between the query and key.

### WRITE ANSWER BELOW ###


Rotary Positional Embeddings (RoPE) encode position information by rotating query and key vectors in the embedding space.

Let:q be the query vector and k be the key vector.

Suppose token positions are: m and n respectively.


Step 1: Apply Rotary Embedding

For position m: qm=R(mθ)q

For position n: kn=R(nθ)k

where: R(⋅) is a rotation matrix.


Step 2: Compute Attention Score

Self-attention score is:Smn=qmTkn
	​

Substituting rotated vectors:Smn=(R(mθ)q)T(R(nθ)k)

Using matrix properties: (Ax)T=xTAT
			 Smn=qTR(mθ)TR(nθ)k


Step 3: Use Rotation Matrix Property

For rotation matrices:R(α)T=R(−α)

Therefore: Smn=qTR(−mθ)R(nθ)k and R(−mθ)R(nθ)=R((n−m)θ)

Hence: Smn=qTR((n−m)θ)k


Final Result

The attention score becomes: Smn=qTR((n−m)θ)k

Notice that the score depends only on: (n−m)


which is the relative positional difference between the key and query positions.

Therefore, in Rotary Positional Embeddings, the self-attention score depends on the distance between tokens rather than their absolute positions.

This property enables transformers using RoPE (such as LLaMA) to model relative positions naturally and generalize better to longer sequences.
